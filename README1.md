<Specification.md (수정 및 확장 버전 — Console 기반 CampusPlanner)>


1. 목적

본 문서는 CampusPlanner 프로그램의 “무엇을(What)” 구현할지를 정의한다.
이 프로젝트는 Android/Firebase 기반이 아닌, Java 콘솔 환경 + Local File 저장 방식으로 동작하는 일정 관리 시스템이다.


2. 프로젝트 개요

CampusPlanner는 학생들이 할 일(To-Do), 시간표, 다이어리, 친구 목록, 성적 정보를 콘솔에서 효율적으로 기록하고 관리할 수 있도록 지원하는 데스크톱(Java) 애플리케이션이다.

GUI(Android Studio X) 대신 Console Text UI

클라우드(Firebase Firestore X) 대신 로컬 파일(program_data.bin) 저장

알림(NotificationManager X) 대신 ScheduledExecutorService 기반 콘솔 알림



3. 핵심 기능 목록 (Console 버전에 맞게 반영)
   
A. 할 일(Task) 관리
| 코드 | 기능명 | 설명 |
|------|--------|------|
| A1 | 할 일 추가 / 수정 / 삭제 | 제목, 마감일, 메모를 입력하여 저장 및 삭제 가능 |
| A2 | 완료 상태 변경 | true/false 또는 콘솔 입력으로 완료/미완료 전환 |
| A3 | 마감 임박 확인 | 현재 시간 기준 특정 기간 내 마감할 할 일만 출력 |
| A4 | 콘솔 알림 | ScheduledExecutorService 로 마감 시간 시 자동 출력 알림 |


B. 시간표(Schedule) 기능
| 코드 | 기능명 | 설명 |
|------|--------|------|
| B1 | 수업 추가/수정/삭제 | 과목명, 강의실, 요일, 시작/종료 시간 입력 및 삭제 가능 |
| B2 | 중복 수업 방지 | 같은 요일/시간대 중복 수업 등록 시 거부 |
| B3 | 요일별 조회 | MON~SUN 또는 '월~일' 입력 시 해당 요일 시간표 출력 |
| B4 | 다음 수업 확인 | 현재 시간 기준 가장 가까운 수업을 자동 계산 및 출력 |

C. 성적(Grade) 관리
| 코드 | 기능명 | 설명 |
|------|--------|------|
| C1 | 성적 추가 | 특정 과목의 점수(0~100)를 등록 |
| C2 | 성적 수정/삭제 | ID 기반 성적 수정 또는 제거 가능 |
| C3 | 평균 점수 계산 | 전체 과목 점수의 평균 계산 |
| C4 | GPA 계산 | 평균 점수를 A+~F 등급 및 4.5점 만점 GPA로 환산 |
| C5 | 성적 요약 | 최고점, 최저점, 평균, 과목 수 등 요약 정보 출력 |

D. 캘린더 및 다이어리
| 코드 | 기능명 | 설명 |
|------|--------|------|
| D1 | 다이어리 작성 | 날짜(LocalDate)별 텍스트 메모 저장 |
| D2 | 다이어리 수정/삭제 | 기존 다이어리 내용 변경 또는 삭제 |
| D3 | 날짜별 조회 | 특정 날짜의 다이어리와 할 일 함께 출력 |
| D4 | 이미지 첨부 | Console 버전에서는 이미지 파일 경로 문자열만 저장 |

E. 친구(Friend) 기능
| 코드 | 기능명 | 설명 |
|------|--------|------|
| E1 | 친구 추가 | 이름 + UID 기반으로 친구 정보 저장 |
| E2 | 친구 목록 조회 | 현재 등록된 친구 목록 전체 출력 |
| E3 | 친구 삭제 | 친구 ID 기반 삭제 |
| E4 | 친구 시간표 조회 | (선택 기능) 친구 시간표 열람 기능 |

F. 오늘 보기(Today)
| 코드 | 기능명 | 설명 |
|------|--------|------|
| F1 | 오늘 요약 출력 | 오늘 날짜 기준 할 일 개수 및 일기 개수 요약 |
| F2 | 오늘 마감할 할 일 | LocalDate.now() 기준 오늘 마감 Task 표시 |
| F3 | 오늘 작성된 일기 | 오늘 날짜와 일치하는 Diary 항목 출력 |


4. 비기능적 요구사항 (Console 환경 반영)
| 항목 | 요구사항 (Console 기반) |
|------|---------------------------|
| **UI(사용자 인터페이스)** | 그래픽 UI가 아닌 **콘솔 입력/출력 기반(Scanner + System.out)** |
| **데이터 저장 방식** | **Local File Serialization** (`program_data.bin`) 사용. DB, Firebase 미사용 |
| **저장 안정성** | 저장 시 `.tmp → .bin` 원자적 이동 방식으로 데이터 손상 방지 |
| **성능** | 데이터 저장/불러오기 시간 ≤ 1초, 일반 작업은 즉시 응답 가능 |
| **알림(Notification)** | Android 알림 대신 **ScheduledExecutorService** 기반 콘솔 출력 알림 |
| **보안** | 로그인/인증 기능 없음, 데이터는 PC 내부 파일에 저장됨 |
| **데이터 무결성** | LocalStore에 저장된 객체는 삭제/수정 시 동기화 및 저장됨 |
| **확장성** | MVC + DAO + ViewModel 구조 → Android, GUI(Swing/JavaFX)로 확장 가능 |
| **오프라인 사용 가능** | 네트워크 연결 없이도 모든 기능 동작 (Local 환경 완전 독립형) |
| **의존성** | 외부 라이브러리 없음 (표준 Java 라이브러리만 사용) |
